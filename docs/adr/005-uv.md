# ADR 005: `uv` for Python Package Management

## Status
Accepted

## Context
Python environments and dependency resolution have traditionally been managed using `pip`, `pipenv`, or `poetry`. These tools can be slow, especially in CI/CD environments and multi-stage Docker builds.

## Decision
We will use **`uv`** (by Astral) as our Python package installer and environment manager in Docker.

## Consequences
- **Positive**: `uv` is written in Rust and is 10-100x faster than `pip`. 
- **Positive**: It provides deterministic lockfiles, ensuring reproducible builds.
- **Positive**: It supports excellent Docker layer caching, significantly reducing container build times.
- **Negative**: It is a newer tool, meaning some edge-case packages might have issues, but for standard FastAPI/ML dependencies, it is fully production-ready.
